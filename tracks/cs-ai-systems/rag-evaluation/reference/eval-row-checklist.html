Eval Row Checklist

Use this before adding a question to the PMP-docs RAG evaluation dataset.

The Contract Corpus

The first dataset should use only these source files unless the module contract is intentionally updated:

AGENTS.md

Repo operating rules, agent roles, validation command, and artifact-level definitions.

PROGRAM.md

Program mission, positioning, principles, budget, evidence model, assessment model, and first milestone.

ROADMAP.md

Current phase, active track, candidate modules, future tracks, and next decision.

CONTEXT.md

Domain vocabulary for program, track, module, evidence, artifact levels, source of truth, and weekly review.

PROGRESS.md

Current program status, active module, module ledger, and weekly review table.

ARTIFACTS.md

Public proof index, artifact levels, and indexed artifacts.

Four Checks

Check Passes when Common repair
Corpus fit Every expected source is inside the contract corpus. Use a contract source, or mark this as an intentional corpus expansion.
Atomic facts Each expected fact is one checkable claim, not a theme. Split broad phrases into concrete facts.
Source auditability A human can find the fact in the named source without guessing. Add the right source or narrow the question.
Failure usefulness A wrong answer would point to retrieval, generation, dataset ambiguity, or missing source. Make the expected facts and sources more specific.

A good row is small enough to judge by hand. If you cannot audit it manually, it will not become clearer after adding metrics.

Good Row Shape

{
  "id": "pmp-002",
  "question": "What must every PMP module produce?",
  "expected_facts": [
    "Every module must produce evidence.",
    "Not every module must produce polished public proof."
  ],
  "expected_sources": ["PROGRAM.md"],
  "retrieved_chunks": [],
  "actual_answer": "",
  "failure_labels": []
}

Avoid facts like "talks about evidence" or "listing out ideas". Those are topics, not audit targets.

Source-Backed Metric Link

RAG metrics such as context recall and answer correctness only make sense when expected facts and expected sources are explicit. For vocabulary, pair this checklist with the RAG evaluation glossary and the DeepEval RAG evaluation guide.
