Lesson 0003

Run a Manual RAG Diagnostic

This is where eval rows start paying rent: each row becomes a tiny debugger for the assistant you want to build.

Recommended sequence: Lesson 0000, Lesson 0001, Lesson 0002, then this lesson.

The bigger picture

Your target system is a PMP operating assistant: it should understand project rules, create modules correctly, track progress, and help you maintain the learning program. The rows from lesson 2 are not paperwork. They are probes you send through the system to find out what breaks.

Primary source: the DeepEval RAG evaluation guide, which separates retriever metrics from generator metrics. For this lesson, do that split by hand first using the manual diagnostic loop.

One row becomes one experiment

{
  "id": "pmp-ops-001",
  "question": "Create a new PMP module. What must happen before normal /teach lesson work begins?",
  "expected_sources": ["AGENTS.md"],
  "expected_facts": [
    "MODULE_CONTRACT.md must exist before normal /teach lesson work begins.",
    "If MODULE_CONTRACT.md is missing, the agent should interview briefly and create it from templates/MODULE_CONTRACT.md.",
    "After the module contract exists, the agent should create or update MISSION.md, RESOURCES.md, lessons/, reference/, learning-records/, and NOTES.md."
  ]
}

If the assistant fails this row, you do not yet know whether the system is bad. You only know this case failed. The diagnostic loop tells you what to inspect next.

Simulated run

Retrieved chunk A

ROADMAP.md: Current phase goals include choosing the first module sequence and starting the first module with a module contract.

Retrieved chunk B

PROGRAM.md: Modules should be small enough to complete in weeks, not semesters.

Actual answer: "Choose a module, write its contract, then begin lessons."

The answer is plausible, but the retrieved chunks do not include the detailed /teach overlay rules from AGENTS.md. The assistant cannot be trusted to create the right files yet.

Practice

What is the best diagnosis for the simulated run?

Choose one diagnosis to get feedback.

Why this matters

Without the row

"The assistant is bad at creating modules." That is emotionally clear and technically useless.

With the row

"The corpus contains the rule in AGENTS.md, but retrieval did not fetch it." Now you know to inspect retrieval before changing the prompt.
