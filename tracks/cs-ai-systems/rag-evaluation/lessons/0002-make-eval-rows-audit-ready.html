Lesson 0002

Make Eval Rows Audit-Ready

The next skill is turning a broad user question into a small, source-backed eval row that a human can judge before any metric runs.

Recommended sequence: Lesson 0000, then Lesson 0001, then this lesson.

The win

By the end of this lesson, you should be able to write three dataset rows for the PMP-docs corpus where every expected fact is concrete and every expected source is inside the module contract.

Primary source: the module contract plus the DeepEval RAG evaluation guide. DeepEval's retrieval and generation metrics become more useful when your dataset has explicit expected facts and expected sources. Keep the eval row checklist and glossary nearby.

Your target use case

Your lesson 0 answer gives this module a product shape: a PMP operating assistant that can generate the project from scratch, manage modules, and follow progress asynchronously. Lesson 2 now uses that target, but keeps the first rows inside the contract corpus so the baseline harness stays small.

In this lesson, treat IDEAS.md, templates, and broader docs as future corpus expansion. Start with the source files the contract already names, then add more sources once the first evaluation loop is working.

What changed from lesson 1

Broad topic

"How can you expand the PMP program?" is a real question, but it is too broad until the expected facts say exactly what counts as a correct answer.

Outside corpus

IDEAS.md is useful, but it is not in the first contract corpus. That makes it a corpus-expansion choice, not a default row.

Audit-ready row

A good row names facts like "Every module must produce evidence" and sources like PROGRAM.md.

The row-making loop

  1. Pick one contract source: AGENTS.md, PROGRAM.md, ROADMAP.md, CONTEXT.md, PROGRESS.md, or ARTIFACTS.md.
  2. Choose one useful claim from that file.
  3. Turn the claim into one narrow question.
  4. Split the answer into atomic expected facts.
  5. Ask: if retrieval misses this source, what failure label would I assign?

Worked repair

Original row idea:

{
  "question": "How you can expand the PMP program?",
  "expected_facts": ["listing out ideas on IDEAS.md"],
  "expected_sources": ["IDEAS.md"]
}

Contract-corpus repair:

{
  "id": "pmp-002",
  "question": "What future tracks does the roadmap list?",
  "expected_facts": [
    "The roadmap lists Business and product as a future track.",
    "The roadmap lists Communication and career as a future track."
  ],
  "expected_sources": ["ROADMAP.md"],
  "retrieved_chunks": [],
  "actual_answer": "",
  "failure_labels": []
}

This row is narrower, source-backed, and still captures program expansion.

For your operating-assistant use case, an even more direct row is:

{
  "id": "pmp-ops-001",
  "question": "Create a new PMP module. What must happen before normal /teach lesson work begins?",
  "expected_facts": [
    "MODULE_CONTRACT.md must exist before normal /teach lesson work begins.",
    "If MODULE_CONTRACT.md is missing, the agent should interview briefly and create it from templates/MODULE_CONTRACT.md.",
    "After the module contract exists, the agent should create or update MISSION.md, RESOURCES.md, lessons/, reference/, learning-records/, and NOTES.md."
  ],
  "expected_sources": ["AGENTS.md"],
  "retrieved_chunks": [],
  "actual_answer": "",
  "failure_labels": []
}

Practice

Classify this proposed row.

{
  "id": "pmp-003",
  "question": "What does public proof mean?",
  "expected_facts": [
    "Public proof is polished enough to show externally."
  ],
  "expected_sources": ["ARTIFACTS.md"],
  "retrieved_chunks": [],
  "actual_answer": "",
  "failure_labels": []
}
Choose one label to get feedback.

Your task

Write three rows, all inside the contract corpus:

Row A

One row from PROGRAM.md about evidence, assessment, budget, or the first milestone.

Row B

One row from CONTEXT.md about a domain term such as module, source of truth, validated artifact, or weekly review.

Row C

One row from AGENTS.md about correctly handling a "create a new module" request.

Use this JSON shape:

{
  "id": "pmp-00X",
  "question": "",
  "expected_facts": [],
  "expected_sources": [],
  "retrieved_chunks": [],
  "actual_answer": "",
  "failure_labels": []
}

Send the three rows back to the agent for feedback. If they are audit-ready, the next step is to turn them into the first tiny dataset file.
