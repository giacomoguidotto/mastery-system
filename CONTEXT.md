# Context

This glossary defines the domain language for the Personal Mastery Program.

## Program

The top-level training system. It owns sequencing, progress review, public proof, and cross-track decisions.

## Track

A coherent area of study inside the program, such as CS/AI Systems or Business and Product.

## Module

A small `/teach` workspace with one mission, one module contract, source-backed lessons, and evidence. Modules should usually fit into 8-20 focused hours, with flexibility when the work genuinely needs more time.

## Module Contract

The PMP-specific file that defines a module's capability target, prerequisites, outputs, feedback mix, completion rubric, and public artifact target before `/teach` lessons begin.

## Evidence

Rough proof that learning happened. Evidence can be private-looking, messy, and still useful.

## Validated Artifact

Evidence that has been checked against tests, sources, rubrics, benchmarks, expert material, or review.

## Public Proof

A polished artifact suitable for external readers: repo, demo, essay, benchmark, reproduction, pull request, architecture note, or project milestone.

## Source of Truth

Named external authority for a module: textbook, university course, paper, official docs, source code, benchmark, or production behavior. Agents are not sources of truth.

## Weekly Review

The program-level review ritual. It starts from Git history and updates progress, artifacts, roadmap, and next-week allocation.
