## Decision

Personal Mastery Program will live as a public GitHub repository. The repo is first Giacomo's training system and second a reusable public artifact.

## Rationale

The program does not produce an accredited credential, so it needs visible evidence of capability. Keeping the work public creates an audit trail, makes progress legible, and may help others reuse the structure later. The program must still optimize for real capability gain before audience growth.

## Consequences

- Program-level docs live in the repo.
- Module work can be public by default.
- Every module must produce evidence.
- Not every module needs polished public proof.
- Agents should preserve the distinction between raw learning work, validated artifacts, and public proof.
